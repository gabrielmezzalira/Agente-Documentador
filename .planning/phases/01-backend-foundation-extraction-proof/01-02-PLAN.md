<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> /ingest | Untrusted file bytes + form fields cross here |
| /ingest -> extraction_graph | Validated MIME-checked payload enters the graph |
| extraction_graph -> Gemini API | texto_preprocessado leaves the system to a third-party LLM |
| extraction_graph -> Supabase | Extracted dict written as jsonb |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service | Large file upload | mitigate | preprocessar_arquivo truncates at 50_000 chars before Gemini; truncation logged (AI-SPEC failure mode #5 / Security Domain DoS row) |
| T-02-02 | Tampering | Unsupported file types reaching the graph | mitigate | D-02 router-level MIME check rejects non-text/plain with 422 before graph invocation |
| T-02-03 | Tampering | Malformed/fabricated JSON into ingestions jsonb | mitigate | with_structured_output(ConteudoEstruturado) Pydantic-validates before insert; completeness guardrail blocks empty rows (AI-SPEC section 6) |
| T-02-04 | Information Disclosure | Full document content in logs/traces | mitigate | Log only first 200 chars of texto_preprocessado; never log GEMINI_API_KEY (AI-SPEC section 7) |
| T-02-05 | Information Disclosure | Silent save failure reported as success | mitigate | salvar checks response.data; failure -> valido=False -> HTTP 502 so manager knows data was NOT saved (AI-SPEC failure mode #4, online guardrail) |
| T-02-06 | Repudiation | No record of which file produced which extraction | accept | ingestions stores file_name + sprint_number + created_at; sufficient for MVP audit trail |
</threat_model>
